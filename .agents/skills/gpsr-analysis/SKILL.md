# GPSR (Goals-Problems-Solutions-Risks) Framework

## Trigger: Project Type Reference
Invoke this for all new projects except Type (a).
- **Type (a) Learning:** Skip GPSR; Light GPSR depth.
- **Type (b) Self-Tool:** Moderate GPSR depth.
- **Type (c) For Others:** Full GPSR depth.
- **Type (d) Commercial:** Full GPSR + Market/Stakeholder context.

## Core Analysis Process
1. **Elicit Goals:** Define outcomes in a strict hierarchy.
2. **Identify Problems:** Map causal barriers to goals.
3. **Propose Solutions:** Classify as **Adopt → Configure → Extend → Build**.
4. **Identify Risks:** Potential consequences of solutions with mitigation plans.
5. **Define Scope:** Explicitly state what is IN and OUT.

## Traceability Rules (Completeness Criteria)
- Every goal has at least one solution or sub-goal.
- Every problem traces to at least one goal it obstructs.
- Every solution traces to a problem it resolves or a goal it advances.
- Every risk has a mitigation plan.

## Causal Defensibility Checks
- **Execution Test:** Does solution X plausibly cause sub-goal Y?
- **Neutralization Test:** Does sub-goal Y actually eliminate problem Z?
- **Completeness Test:** Are sub-goals collectively sufficient for the goal?

## Output: GPSR Analysis Document
1. Goal Hierarchy
2. Problem Map
3. Solution Inventory (Classify: Adopt/Configure/Extend/Build)
4. Requirements Baseline (Living Specification)
5. Risk Register
6. Scope Statement
7. Alignment Diagram (Directed Graph)
