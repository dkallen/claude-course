Module 4: Quick Reference

Use this when choosing a Codex permissions mode for real work.

Core Distinction

SandboxDefines the technical boundary.
Approval policyDefines when Codex must stop and ask.

Common Modes

Mode / PolicyUse it for
read-onlyInspection, diagnosis, explanation
workspace-writeNormal local editing work
danger-full-accessIntentional full-trust operation only
on-requestWork normally inside the boundary; ask when it must cross it
untrustedGate commands outside the trusted set
neverNon-interactive runs where prompts must not appear

Remember

  • --full-auto is workspace-write plus on-request.
  • True full access is danger-full-access plus never.
  • Network and outside-workspace writes are separate boundary questions.
  • Prefer writable roots or rules to broad access when a task needs one exception.
