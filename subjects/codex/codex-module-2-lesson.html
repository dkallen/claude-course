Module 2: AGENTS.md — Teaching Codex How You Work

Working Effectively with OpenAI Codex — A Practitioner's Course

Why This Module Matters

Codex gets much better when the repository itself teaches it how to work. OpenAI’s docs are explicit here: `AGENTS.md` gives Codex durable project guidance that travels with the repository and applies before the agent starts work.

Without that file, you are forced to restate the same expectations in prompt after prompt. With it, the repository starts carrying part of the working relationship.

1. What AGENTS.md Is

Think of `AGENTS.md` as an open-format README for the agent. It is not product documentation for humans first. It is repository guidance for Codex first.

OpenAI’s guidance recommends using it for:

  • repo layout and important directories
  • how to run the project
  • build, test, and lint commands
  • engineering conventions and review expectations
  • constraints and do-not rules
  • what done means and how to verify work

2. Discovery and Precedence

Codex builds an instruction chain when it starts. According to the current docs, it can read a global file under ~/.codex, then project-level files, then more specific files closer to the current working directory. Files later in that chain override earlier guidance.

The practical meaning is simple: broad defaults globally, shared norms at repo root, and specialized rules near specialized code.

3. Keep It Small

OpenAI’s docs say to keep `AGENTS.md` practical and small. That is not a stylistic preference. It is an accuracy preference. Bloated guidance competes with itself.

Start with only the instructions that matter. Add new rules only after you see repeated mistakes or repeated review feedback.

4. What Belongs There vs. In a Prompt

Put durable repository truth in `AGENTS.md`:

  • use npm test after touching frontend code
  • treat public API changes as doc changes too
  • follow this review checklist
  • never add production dependencies without confirmation

Put one-off task details in the prompt:

  • fix this exact bug
  • work only in these files
  • review against this branch

5. Use It as a Feedback Loop

One of the best lines in the current docs is the idea that when Codex makes the same mistake twice, you should update `AGENTS.md`. That is the right operational model. The file is not a sacred artifact. It is a memory of recurring friction.

6. Why This Matters More Than It First Appears

Good `AGENTS.md` files improve more than raw prompting. They also improve testing, review quality, and consistency across the app, IDE, and CLI. They are one of the main ways Codex stops feeling generic.

Check

If you can explain what belongs in `AGENTS.md`, how Codex discovers it, and how to use it as a feedback loop, you are ready to start writing one for a real repo.
