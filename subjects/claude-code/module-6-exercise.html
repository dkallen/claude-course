Module 6: Guided Exercise

Run one complete red-green-refactor cycle with Claude Code on a real change.

Goal: Use tests as specification before implementation begins.

Time estimate: 35–45 minutes

Part 1: Pick a Small Real Change

Choose one bug fix, one validator, or one focused behavior change. If the task is broad, TDD becomes muddy.

Part 2: Write Requirements in English

Write 3 to 5 short statements describing what the software should do. Include at least one edge case.

Part 3: Ask Claude for Tests Only

Tell Claude to generate tests from the requirements and to stop before implementation. Review the tests before you let it write code.

Part 4: Review the Tests Ruthlessly

  • Do they fail if the feature is missing?
  • Are the assertions about behavior rather than implementation detail?
  • Is there at least one edge case that matters?
  • Would these tests catch a real regression?

Part 5: Confirm Red

Run the tests and make sure they fail for the intended reason. If they already pass, the loop is broken.

Part 6: Ask Claude for the Smallest Green

Now ask Claude to implement the smallest coherent change that makes the tests pass. Do not combine this with refactoring yet.

Part 7: Refactor Deliberately

Once the tests are green, ask Claude to improve structure without changing behavior. Rerun the tests.

Part 8: Capture the Result

Your final artifact should include:

  • plain-English requirements
  • the failing tests
  • the minimal implementation
  • the refactor pass
  • one note on what the tests protected that prose alone would not
