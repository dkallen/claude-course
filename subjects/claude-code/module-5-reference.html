Module 5: Quick Reference

Use this checklist when turning a brief into architecture.

Ask

  • What major responsibilities exist?
  • What should each component own?
  • What interfaces matter most?
  • What tradeoffs are we accepting?
  • What risks is this design managing?

Avoid

  • starting from preferred tools instead of the brief
  • vague layers with no clear job
  • duplicated business rules across boundaries
  • pretending first-pass architecture should settle every detail
