Prompt Spectrum

Same task, different prompt shapes. Use this to see how the framing shifts from exploration to execution.

Loose

Coffee-Chat Prompt

Best for exploring a problem and seeing what Claude notices.

This login flow feels messy. Can you take a look and clean it up?
Outcome-First

Director Mode

Best when you know the outcome but don't want to micromanage the implementation.

Refactor the login flow so validation lives in one place, the UI stays the same, and the change is easy to review.
Structured

Goal + Constraints

Best when scope control and verification matter.

Goal: centralize login validation. Constraints: preserve UI, no new dependencies, follow existing patterns. Done when: tests pass and validation logic is no longer duplicated.
Over-Specified

Micromanaged

Useful rarely. Can block Claude from taking a cleaner path based on what it finds.

Open file A, move lines 20–60 into file B, rename the function to X, then edit file C, then update imports manually...
