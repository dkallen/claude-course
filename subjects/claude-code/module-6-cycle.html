Red-Green-Refactor Map

Read the cycle as a control loop. The requirement becomes a test, the test constrains implementation, and refactoring happens only after behavior is protected.

Red

The test should fail because the behavior is missing, not because the setup is broken or the assertion is meaningless.

Green

Ask Claude for the smallest coherent implementation. Do not combine implementation and cleanup in one jump.

Refactor

Only refactor after the tests are green. Otherwise the loop loses its safety signal and turns back into guess-and-check.
