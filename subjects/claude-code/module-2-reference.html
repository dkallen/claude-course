What is CLAUDE.md?

CLAUDE.md is a plain-text Markdown file with persistent instructions for Claude Code. It's automatically read at the start of each session—no setup required—and survives across sessions (unlike conversation history). Use it to establish conventions, patterns, and team standards that should apply continuously to your project.

File Locations & Hierarchy

CLAUDE.md can exist at multiple levels. More specific files override more general ones.

Level Path Scope Use For
Global ~/.claude/CLAUDE.md All projects Personal preferences, coding style
Project ./CLAUDE.md This project Tech stack, team conventions, architecture
Folder ./src/CLAUDE.md This folder Specialized rules (tests, API, etc.)
Override rule: Folder rules override project rules override global rules.

CLAUDE.md vs. Prompt — Quick Decision

Put in CLAUDE.md

  • Coding conventions
  • Framework patterns
  • Architecture decisions
  • Team standards & SOPs
  • Analysis frameworks
  • Code style rules

Put in Prompt

  • Current task
  • Feature requirements
  • One-off instructions
  • Session-specific context
  • Temporary modifications
  • User questions

Writing Effective Instructions

Best Practices

  • Be specific and actionable. Not "write good code," but "use 2-space indentation and add JSDoc for exports."
  • Include examples when helpful. Show what you mean.
  • State what TO do, not just what NOT to do. "Use async/await" is better than "don't use callbacks."
  • Include your analysis frameworks as standard operating procedures.
  • Keep it focused. Quality over quantity—concentration matters.

Anti-Patterns (Avoid These)

  • Too long (500+ lines): Instructions compete for attention. Prioritize ruthlessly.
  • Too vague: "Be careful," "write clean code." Not actionable.
  • Contradictory: "Always test" + "skip non-essentials." Confuses Claude.
  • Task-specific content: Belongs in prompts, not CLAUDE.md.
  • Never updating: Review periodically as your project evolves.

.claudeignore

Works like .gitignore for Claude. Use it to keep build artifacts, generated files, logs, dependencies, and sensitive data out of context window:

  • node_modules/
  • dist/
  • .env, .env.local
  • *.log
  • Generated or compiled files
