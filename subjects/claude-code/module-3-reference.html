Module 3 Quick Reference

Default Prompt Structure

  • Goal: what should change
  • Context: what Claude needs to inspect
  • Constraints: what must not change
  • Done when: how finished work is judged

Use Loose Prompts When

  • you are diagnosing or exploring
  • the problem is not fully framed yet
  • you want Claude to help define the shape of the work

Use Tight Prompts When

  • the task is high-stakes
  • scope drift would be expensive
  • you need a result that is easy to verify and review

Fast Heuristic

  • Am I exploring or executing?
  • What context does Claude actually need?
  • What must not change?
  • How will I know the task is done?
