Module 6: Quick Reference

Use this when you want Claude to work test-first instead of coding first.

The Cycle

StageQuestion
RedWould this test fail if the behavior did not exist?
GreenWhat is the smallest change that makes it pass?
RefactorWhat can be cleaned up now that behavior is protected?

Review Tests Before Implementation

  • Does each test validate behavior instead of internals?
  • Is there at least one meaningful edge case?
  • Would the tests catch a real regression?
  • Do they fail for the intended reason before implementation exists?

Choose The Test Level

Use unit tests whenUse integration tests when
The risk is in isolated logic and you want fast feedback.The risk is in the interaction between parts, layers, or services.

Prompt Skeleton

  • Write tests only from these requirements.
  • Do not implement yet.
  • Make the first tests fail for clear reasons.
  • After review, implement the smallest change to go green.
