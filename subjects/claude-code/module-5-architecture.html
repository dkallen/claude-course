Module 5: Architecture Map

A compact view of how the refined brief becomes a buildable structure.

Input

Start with the Module 4 brief: goals, problems, aligned solutions, risks, and scope.

Brief -> components -> interfaces -> tradeoffs -> implementation plan

Components

  • name the major parts
  • give each part one clear job
  • separate business rules from delivery mechanics

Interfaces

  • what data crosses boundaries?
  • what promises do parts make to each other?
  • what can change locally without breaking the rest?

Tradeoffs

  • simplicity vs flexibility
  • speed now vs adaptability later
  • isolation vs coordination cost

Output

A short architecture brief with named components, responsibilities, interfaces, risks, and open questions.
