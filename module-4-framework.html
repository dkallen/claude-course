Framework Map

This directed graph shows the core logic of the module. Read arrows as verbs: a problem obstructs a goal, a solution resolves a problem, a risk threatens a solution, and a mitigation solution mitigates that risk.

The graph is directional on purpose. A node matters because of the relationship it has to something above it or against it. If a solution points nowhere useful, it is probably misaligned. If a risk does not threaten a specific solution, it is probably too vague.

Goal

The desired state. Put it at the top so everything else has to justify its relationship to that outcome.

Problem

A current obstacle. It belongs directly beneath the goal it blocks.

Solution

An action that resolves a problem or directly advances a goal. The arrow must point somewhere defensible.

Risk

A forecasted problem caused by a chosen solution. It points against the solution it threatens.

Mitigation

A second solution aimed at reducing the probability or impact of the risk.

Read The Graph Like This

  • Ask what specific goal the problem is obstructing.
  • Ask whether the proposed solution actually resolves that problem.
  • Ask what new risk the chosen solution introduces.
  • Ask whether the mitigation is worth the added complexity.
